Stage 3 · learned features on top of the backbone

Lags, holidays, promotions, oil, earthquake

Stage 3 adds the features that depend on past sales or external context — everything Stage 2's date-only backbone cannot see. Every feature has to clear the no leakage and lag ≥ 16 rules.

Source: per-feature files under analyze/concepts/* and analyze/data-traps/*.

Stage 3 · short-horizon booster

Lag & rolling features

Per series: lag_16, lag_17, …, and rolling windows anchored at shift(16) (e.g. 7-day rolling mean of sales as of 16 days ago). Same shape, different statistic — they tell the model "what level was this series at, 16+ days ago." For transactions (Trap 6), lag-only — never same-day.

Why ≥ 16? Spelled out in Stage 2 · lag horizon.

Stage 3 · the effective calendar

Holiday features

Build the effective calendar from holidays_events.csv: drop the 12 transferred = True originals, honor the matching Transfer / Bridge / Work Day rows, scope by locale (National → all stores, Regional → that state, Local → that city). Output one or more clean flags per row. Don't shove the raw rows in — see Trap 3 for why.

Stage 3 · payday wave

Payday features (a wave, not a flag)

From Stage 0: the naive "is it the 15th?" flag is ≈ 1.01× (negligible), but a real ≈ 1.24× cycle exists across the month. Encode days-since-payday or a short post-payday window — that lets the model express the wave smoothly.

Stage 3 · the easy one

Promotion feature

onpromotion is delivered same-day in the test horizon (no nulls), so we feed it directly to the model — no lag. It has no signal before 2014-04-01, so don't design features that assume it has always existed.

Stage 3 · oil

Oil feature (forward-filled, modest expectations)

Forward-fill first (no interpolation — that peeks at the future, see Trap 2). Use as a feature, but remember Stage 0's bonus result: the raw oil↔sales correlation of −0.62 collapses to ≈ 0.05 after detrending. Don't expect a miracle; the holdout decides.

Stage 3 · label, don't delete

Earthquake-window flag

A deliberately generous binary (or decaying) flag spanning ≈ 2016-04-16 → mid-May 2016. From Stage 0: the acute spike (≈ 1.41×, peak 1.76× on Apr 18) is only ~1 week — but a generous window is fine; the model can downweight the tail.

Stage 3 in one sentence

Every new column is knowable at prediction time: lags reach back ≥ 16 days; oil is forward-filled; transactions are lag-only; promo is the leak-free same-day signal. assert_no_leak() raises the moment you forget.

Where: src/features.py (every feature builder composes through here).
