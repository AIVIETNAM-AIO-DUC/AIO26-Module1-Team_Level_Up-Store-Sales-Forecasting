Stage 1 · the floor any model must beat

Forecast baselines: seasonal-naive & near-zero

The simplest possible ways to predict — no training, no maths. Stage 1's job is to score these on the holdout and record the number to beat. They also serve as the fallback rules for series too thin to model (see Trap 5).

Source: analyze/concepts/baselines.md.

Stage 1 · two rules, no training

The two baselines

Both predict the future 16 days; neither changes past data.

how it works Seasonal-naive = copy last week forward

Naive = don't learn anything; copy the past forward. Seasonal = copy from the same point in the previous cycle (here, the weekly cycle).

predict next Monday  = last Monday's sales
predict next Tuesday = last Tuesday's sales
…repeat last week's 7-day pattern across the 16-day horizon

Sample data — store 1, GROCERY I (real)

The last known week (7 days ending Tue Aug 15, 2017):

Last weekWed 8/9Thu 8/10Fri 8/11Sat 8/12Sun 8/13Mon 8/14Tue 8/15
sales271925911270163095224072508

Seasonal-naive copies each weekday forward — the 7-day pattern just repeats across the horizon, preserving this series' own shape (here a Sunday dip to 952 against midweek highs of ~2.5–2.7k).

how it works Near-zero = "sells nothing → predict nothing"

If a series has essentially no sales ever, the safest forecast is ≈ 0. Example — store 1, BOOKS, whose last week is all zeros:

Last 7 days (store 1, BOOKS)0000000

→ predict 0 for all 16 horizon days. Nothing to copy.

Stage 1 · routing

Who decides which rule? (not the model)

A common misconception: the trained model does not choose between the main prediction, seasonal-naive, and near-zero. The choice is made before modeling by a simple rule-based gate (an if check) on each series' history. Thin series are peeled off and never reach the model.

For each of the 1,782 series:
   ├─ Has enough history?  ──►  use the MAIN model (the learned one)
   └─ Too thin?            ──►  use a FALLBACK rule
                                   ├─ a little history to copy  ►  seasonal-naive
                                   └─ basically no sales ever   ►  near-zero (~0)

A trained model is a function: it always returns a number — it can't output a blank. The risk for a no-history series isn't emptiness; it's that the number is unreliable garbage. So we route thin series around the model (the cleaner option), train only on healthy ones, then stitch both prediction sets back together into all 28,512 rows — done in Stage 4.

Where: the fallback path in src/models.py; scored by rmsle() on the holdout, exactly like every later stage.
